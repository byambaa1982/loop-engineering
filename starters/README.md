# Starters

Starter loops are small operating contracts for common engineering workflows. Use `loop init --pattern <slug>` to create the first set of repository documents for a loop.

Recommended first patterns:

- `daily-triage` for issue review and routing.
- `pr-babysitter` for pull request follow-up.
- `dependency-sweeper` for low-risk package update batches.
- `post-merge-cleanup` for documentation and release-note hygiene.
