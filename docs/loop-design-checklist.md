# Loop Design Checklist

Use this checklist before running a new agent loop repeatedly.

- The loop has a named owner.
- The trigger is specific and observable.
- The allowed files and systems are clear.
- The agent has a stop condition.
- The output format is easy to review.
- The loop has a token and human-review budget.
- The rollback path is written down.
- The run log captures result, cost, and reviewer.
- The first runs are report-only or reviewed.
